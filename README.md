# AAPL翌日リターン方向予測 — AWS SageMakerパイプライン

株価の翌日リターンの方向（上昇/下落）を予測する分類モデルを、AWS上で完結する形で構築した個人プロジェクト。「予測モデルを正しい方法論で構築・評価できること」と「クラウド上でMLワークフローを運用できること」を目的とする。
## アーキテクチャ

![architecture]

| 役割 | サービス |
|---|---|
| データ保存（特徴量・モデル・評価結果） | Amazon S3 |
| 学習・評価環境 | Amazon SageMaker Notebook（ml.t3.medium） |
| 権限管理 | AWS IAM（SageMaker実行ロール） |
| 自動再学習| AWS Lambda + Amazon EventBridge |

パイプライン: ローカルでデータ取得 → S3にアップロード → SageMakerがS3から読込・学習 → モデル/評価結果をS3へ保存 → コードをGitHubで公開

## データと特徴量

- 対象: Apple (AAPL) 日次株価、約1000営業日
- 特徴量 21個（すべて t日終値までの情報のみ使用）
  - リターン系: `return_1d/5d/21d/63d/252d`、直近4日のラグ
  - モメンタム系: `momentum_5d/21d/63d/252d` と期間間の差分
  - ボラティリティ系: `volatility_5d/21d/63d/252d`
  - その他: `presidential_cycle`（米大統領選サイクル:大統領選期間は株価の変動の影響を受けやすいため、その期間を除く。）
- ターゲット: t+1日のリターンが正なら1（`return_1d.shift(-1) > 0`）

## 方法論（リーク防止を最優先）

1. **ターゲットのシフト**: t日の特徴量で t+1日の方向を予測。
2. **TimeSeriesSplit（n_splits=5, gap=1）**: 通常のKFoldは未来のデータで学習して過去を評価するリークを起こすため不使用。ターゲットがt+1日のため、訓練末尾とテストの間に1日のgapを設定
3. **強いベースラインとの比較**:
   - `naive_up`: 常に「上昇」と予測（株式は上昇日がやや多く、意外と強い）
   - `persist`: 前日と同じ方向を予測
4. **評価指標**: accuracy / AUC / F1 に加え、予測に基づくロング・ショート戦略の**Sharpe比**をbuy & holdと比較

## モデル

- ロジスティック回帰（StandardScaler + C=0.1）
- XGBoost（n_estimators=200, max_depth=3, learning_rate=0.05）

## 考察

- **XGBoostはロジスティック回帰を全指標で上回った**（acc +3.3pt、AUC +2.1pt、Sharpe -0.15→0.19）。非線形・交互作用の捕捉による改善と考えられる
- しかし**どちらのモデルもnaive_up（常に上昇: 0.540）に勝てず**、AUCも0.5近辺。現在の価格由来特徴量には翌日方向の情報がほぼ含まれていない
- Sharpe比でもbuy & hold（0.91）が圧倒。fold間のばらつきが大きく、安定した優位性は確認できない
- この結果は効率的市場仮説と整合的であり、**日次リターンの方向予測がいかに難しいかを、リークのない正しい検証で定量的に確認した**ことがこのプロジェクトの成果。誤った検証（KFold、ターゲット未シフト）を行えば容易に「高精度」が出るが、それは実運用では再現しない

## 今後の改善候補

- 予測ホライズンの変更（5日/21日先はノイズが減る。gapもホライズンに合わせ拡大）
- 外部データの特徴量追加（出来高、金利、VIX、USD/JPY）
- `predict_proba`の閾値調整（確信度の高い日のみ取引する戦略）
- `GridSearchCV(cv=TimeSeriesSplit)`によるハイパーパラメータ調整
- Lambda + EventBridgeによる日次の自動データ更新・再学習

## 再現手順

1. S3バケットを作成し、`feature_dataset.csv`をアップロード
2. SageMaker Notebookインスタンス（ml.t3.medium、実行ロールにS3アクセス権）を作成
3. `notebooks/baseline_model.ipynb`を開き、バケット名を自身のものに変更して上から実行
4. 依存: `%pip install matplotlib scikit-learn xgboost`

**コスト注意**: Notebookインスタンスは起動中に時間課金されるため、作業後は必ず停止する（ml.t3.medium ≒ $0.05/時）。
